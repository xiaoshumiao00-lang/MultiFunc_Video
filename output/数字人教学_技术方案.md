# 数字人教学模块 — 技术实现方案

> 版本：v1.0  
> 基于：Pixelle-Video v0.1.15 数字人口播模块  
> 状态：待确认后实施

---

## 一、需求摘要

新增一个**"数字人教学"**模块，核心技术方案与**"数字人口播"**保持一致，并在此基础上扩展：

| 功能 | 要求 |
|------|------|
| 人物形象上传 | 与数字人口播完全一致 |
| 配音合成（TTS） | 与数字人口播完全一致（本地 Edge TTS / ComfyUI 工作流） |
| 服务配置 | 与数字人口播完全一致（RunningHub / 本地 ComfyUI） |
| 生成模式 | 改为上传教学 PPT，支持 `.pptx` 和 `.pdf` 两种格式 |
| 口播文案 | 可选项；未输入时由本地大模型根据 PPT 内容生成；输入时优先使用用户文案 |
| 视频画幅 | 竖屏 9:16（1080×1920）和横屏 16:9（1920×1080） |
| 数字人位置 | 画面右下角，支持位置偏移与缩放控制 |
| 同步要求 | 口型、音频、PPT 内容（含动画节奏）必须同步 |
| 视频时长 | 支持控制最终视频时长 |

## 二、关键前提与约束

### 2.1 当前环境

- 项目为 Windows 便携版，内置 Python 3.11，已安装 `Pillow`、`moviepy`、`ffmpeg-python`。
- **未安装** `python-pptx`、`pymupdf`、`pdf2image` 等文档解析库。
- **未安装** Microsoft Office / LibreOffice，无法直接渲染 PPT 原生动画为视频。
- 数字人口播已使用 ComfyKit 工作流完成"人物图片 + 音频 → 口型同步视频"，该链路可复用。

### 2.2 核心约束

由于无法调用 Office/LibreOffice，**无法直接渲染 PPT 内部复杂动画（如飞入、淡入、路径动画）为原生视频**。

本方案采用**"PPT 页序列 + 文案时间轴 + 简单动效模拟"**的折中方案：
- 将每页 PPT 渲染为高清图片。
- 根据口播文案/LLM 切分，确定每页/每段内容的显示时间。
- 用 FFmpeg 将图片序列组合为与音频等长的背景视频。
- 叠加文字高亮/淡入等简单效果来模拟"按口播逐条出现"的动画感。
- 若用户上传的 PPT 是 PDF 或多页图片，逻辑相同。

> 如需要 100% 还原 PPT 原生动画，需额外安装 LibreOffice 或要求用户预先将 PPT 导出为视频。该选项会在方案中保留为扩展项。

## 四、数据流与生成流程

### 4.1 用户输入

| 参数 | 类型 | 说明 |
|------|------|------|
| `character_assets` | list[str] | 人物图片路径，复用数字人口播 |
| `teaching_file` | str | 上传的 PPT/PDF 文件路径 |
| `teaching_script` | str (可选) | 用户自定义口播文案 |
| `aspect_ratio` | str | `9:16` 或 `16:9` |
| `human_position` | dict | `{x: 0.75, y: 0.75, scale: 0.25, anchor: "bottom-right"}` |
| `duration_mode` | str | `auto`（按音频） / `fixed`（固定时长） |
| `target_duration` | float | `duration_mode=fixed` 时生效 |
| `tts_*` | - | 复用数字人口播全部 TTS 参数 |
| `workflow_path` | dict | 复用数字人口播服务配置 |

### 4.2 生成步骤

```
1. 校验与解析
   ├── 校验 character_assets 非空
   ├── 校验 teaching_file 存在且格式为 .pptx / .pdf
   └── PPTService.parse() → slides[]（每页：index, image_path, text_content）

2. 口播文案生成
   ├── if teaching_script 非空：script = teaching_script
   └── else：调用 LLMService，输入 slides 文本 → 生成教学口播文案

3. 文案时间轴切分
   └── 按"页"或"段落"将 script 切分为 segments[]
       每段：{text, slide_index, start_time, end_time}
       若 duration_mode=fixed，按比例缩放每段时长

4. 音频生成
   └── TTSService(script) → narration.mp3
       同时返回音频时长 audio_duration

5. 背景视频生成
   ├── 按 aspect_ratio 确定目标分辨率（1080×1920 或 1920×1080）
   ├── 将每页 slide 图片缩放并填充到目标分辨率
   └── FFmpeg concat demuxer / filter：按 segments 时间轴生成 background.mp4
       （页与页之间可配置简单转场：cut / fade）

6. 数字人口型视频生成
   └── 复用数字人口播 second_workflow：character + audio → talking_head.mp4

7. 合成最终视频
   ├── TeachingComposer.overlay_human()
   │   ├── 将 talking_head.mp4 按 human_position.scale 缩放
   │   └── 使用 FFmpeg overlay filter 定位到右下角（支持 x/y 偏移）
   ├── 将 background.mp4 与带透明通道/遮罩的人物视频合成
   └── 最终输出 final.mp4

8. 后处理
   ├── 若 duration_mode=fixed 且 audio_duration ≠ target_duration：
   │   └── 调整 TTS 语速 或 截取/循环背景
   └── 写入历史记录
```

## 五、关键技术选型

### 5.1 PPT/PDF 解析

| 格式 | 库 | 用途 | 是否需安装 |
|------|------|------|-----------|
| `.pptx` | `python-pptx` | 读取幻灯片、文本框、备注、动画顺序（有限） | 是 |
| `.pdf` | `pymupdf` (fitz) | 将 PDF 每页渲染为图片，同时提取文本 | 是 |
| 图片兜底 | `Pillow` | 缩放、裁剪、填充背景 | 已安装 |

> 备选：`pdf2image` 需要额外安装 Poppler，不如 `pymupdf` 自包含，优先使用 `pymupdf`。

### 5.2 PPT 动画处理策略

由于无法原生渲染动画，分三级策略：

| 级别 | 策略 | 效果 |
|------|------|------|
| L1（默认） | 每页 PPT 作为一张静态图，按口播时长停留 | 无动画，但内容完整 |
| L2（增强） | LLM 解析每页文本结构，按句子/要点生成"淡入/高亮"时间轴 | 模拟按口播逐步出现 |
| L3（原生） | 如系统检测到 LibreOffice/Office，可调用其将 PPT 导出为视频 | 完整还原动画（扩展项） |

本方案默认实现 **L1 + L2**；L3 作为可配置扩展接口预留，不在首期实现。

### 5.3 口型同步

直接复用数字人口播的 ComfyKit 工作流链路：
- 输入：`character_assets[0]`（人物图片） + `narration.mp3`
- 工作流：`workflows/{source}/digital_combination.json`
- 输出：`talking_head.mp4`（含口型与音频）

### 5.4 视频合成

使用 `ffmpeg-python` 封装 FFmpeg 命令：
- `concat` / `filter_complex`：图片序列 → 背景视频
- `scale` + `overlay`：将数字人视频叠加到背景
- `tpad` / `apad`：时长对齐

## 六、竖屏/横屏适配策略

### 6.1 画幅目标

| 画幅 | 分辨率 | 典型用途 |
|------|--------|----------|
| 竖屏 9:16 | 1080 × 1920 | 抖音/快手/视频号 |
| 横屏 16:9 | 1920 × 1080 | B站/YouTube/课堂 |

### 6.2 PPT 内容适配

PPT 原始比例多为 16:9 或 4:3，与目标画幅可能不一致。提供两种填充模式：

| 模式 | 说明 |
|------|------|
| `fit`（默认） | 等比缩放 PPT 页面，黑边/白边填充，内容完整 |
| `cover` | 等比缩放并裁剪，填满目标画幅，可能损失边缘 |

### 6.3 数字人位置

默认定位右下角，支持用户调整：

```python
human_position = {
    "anchor": "bottom-right",  # 锚点
    "offset_x": 0.02,          # 距离右边 2% 画布宽度
    "offset_y": 0.02,          # 距离底边 2% 画布高度
    "scale": 0.25,             # 人物视频宽度占画布宽度的 25%
}
```

FFmpeg overlay 坐标计算：
- `x = W - w - offset_x * W`
- `y = H - h - offset_y * H`
- 其中 `w = talking_head_w * scale_ratio`

## 七、同步保证方案

### 7.1 音频与 PPT 内容同步

核心思路：**以音频时长为基准，按文案时间轴驱动 PPT 翻页**。

1. 将口播文案按"页"切分（每页对应一段口播）。
2. TTS 生成后，获取每段实际音频时长。
3. 背景视频中每页幻灯片的显示时长 = 对应音频段时长。
4. 若文案未明确分页，由 LLM 根据幻灯片文本智能分配。

### 7.2 口型与音频同步

由 ComfyKit 的数字人工作流保证，与数字人口播一致，不改动。

### 7.3 动画与口播同步

L2 模拟动画时，由 LLM 输出结构：

```json
{
  "segments": [
    {
      "slide_index": 0,
      "text": "大家好，今天讲解...",
      "highlights": [
        {"keyword": "核心概念", "start": 2.5, "duration": 1.2}
      ],
      "animation": "fade_in"
    }
  ]
}
```

TeachingComposer 根据 `start/duration` 在对应时间点叠加高亮框或文字淡入效果。

## 十、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 无法原生渲染 PPT 动画 | 复杂动画丢失 | 采用静态页+模拟动效；预留 Office/LibreOffice 扩展接口 |
| PPT 含特殊字体/矢量图 | 转图片时渲染偏差 | 提醒用户将字体嵌入 PPT；优先使用 PDF 上传 |
| LLM 生成文案与 PPT 不同步 | 口播与画面错位 | Prompt 强制按页输出，并允许用户二次编辑文案 |
| 音频时长与目标时长不一致 | 视频被截断或黑屏 | 自动 TTS 语速微调 + FFmpeg pad/trim |
| 人物视频分辨率与画幅不匹配 | 叠加后变形 | 统一按画布宽度比例缩放，保持人物原始比例 |

## 十一、验收标准

1. 能在 Web UI 中看到"数字人教学"标签页。
2. 支持上传 `.pptx` 和 `.pdf`，并预览幻灯片缩略图。
3. 不输入文案时，能调用 LLM 生成与 PPT 内容相关的口播文案。
4. 生成视频为 1080×1920 或 1920×1080。
5. 数字人出现在右下角，位置和大小可配置。
6. 最终视频音频完整、口型与音频同步、PPT 翻页与口播节奏一致。
7. 支持设置目标视频时长（固定模式）。
